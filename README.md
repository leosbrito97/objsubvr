# Head Metrics Classification Notebooks

This folder is a **paper-reference companion workspace** for the study:

**From Head Motion to Subjective VR Outcomes: A Reproducible Metric Framework and Benchmark on the FAST Dataset**

The intention is the same as in repositories that accompany research papers: provide a reusable, inspectable, and reproducible implementation of the main tabular head-motion workflow used in the study, while also making it easier for other users to adapt the pipeline to their own datasets.

## Relation To The Paper

This `notebook/` folder is a **reference implementation layer** for the paper, not a mirror of every experiment script in the repository.

Its role is to expose, in a cleaner and more portable format:

- the **raw-to-features step** that converts `FAST Scores.xlsx` + `Tracking/` files into questionnaire-specific `HeadFeaturesVS...` spreadsheets;
- the **47-variable schema** used across the experiments: 46 head-motion features plus `vr_system_ordinal`;
- the **classification workflow** based on repeated stratified cross-validation, imbalance handling, and threshold search;
- a compact entry point for adapting the same logic to **new FAST-like datasets**.

This follows the usual style of paper repositories:

- the main repository keeps the full experiment history;
- this subfolder exposes a narrower workflow for reuse and demonstration;
- notebooks provide an interactive path;
- scripts provide the same logic in CLI/Python form.

## Directory Layout

- `data/raw/`
  Raw inputs used by notebook 1. In the bundled demo, this contains a reduced FAST subset with `FAST Scores.xlsx` and matching `Tracking/` CSV files.
- `data/generated/headfeatures/`
  Spreadsheets generated by notebook 1 or by `feature_engineering_pipeline.py`. This includes both the direct `HeadFeaturesVS...` files and derived targets such as `HeadFeaturesVSSUSBinary...`.
- `data/legacy_precomputed_head_metrics/`
  Older demo assets from the previous notebook version, kept only for reference. They are not the active workflow anymore.
- `outputs/`
  Model-selection outputs, metrics, predictions, and serialized artifacts produced by notebook 2.
- `configs/`
  JSON files for feature engineering and classification-pipeline configuration.
- `scripts/`
  Reusable Python code called by the notebooks and CLI commands.

All notebook defaults use paths relative to `notebook/`. This keeps the folder portable when pushed to GitHub or cloned elsewhere.

## Active Workflow

### Notebook 1: feature engineering from raw FAST-style inputs

File:

- `01_prepare_head_metrics_dataset.ipynb`

What it does:

- reads a raw `FAST Scores.xlsx` file;
- reads the corresponding `Tracking/` CSVs;
- matches score rows to tracking files;
- extracts the 46 head-motion features;
- appends `vr_system_ordinal`;
- writes questionnaire-specific `HeadFeaturesVS...` spreadsheets;
- optionally derives a modeling dataset such as `SUS binary`.

Default demo outputs:

- `data/generated/headfeatures/HeadFeaturesVSSUS_BuildA.xlsx`
- `data/generated/headfeatures/HeadFeaturesVSSUS_BuildB.xlsx`
- `data/generated/headfeatures/HeadFeaturesVSSUSBinary_BuildA.xlsx`
- `data/generated/headfeatures/HeadFeaturesVSSUSBinary_BuildB.xlsx`

### Notebook 2: configurable classification pipeline

File:

- `02_run_classification_pipeline.ipynb`

What it does:

- loads a generated train spreadsheet and a generated test spreadsheet;
- runs repeated stratified CV only on the train side;
- compares multiple models, preprocessing strategies, imbalance strategies, and threshold strategies;
- selects the best pipeline on the train side;
- evaluates once on the test side.

The default demo is `SUS binary`, using:

- train: `HeadFeaturesVSSUSBinary_BuildA.xlsx`
- test: `HeadFeaturesVSSUSBinary_BuildB.xlsx`

## Bundled Demo Data

The repository now includes a **raw demo subset**:

- `data/raw/fast_demo/FAST Scores.xlsx`
- `data/raw/fast_demo/Tracking/`
- `data/raw/fast_demo/demo_participants_manifest.csv`

This is a reduced FAST subset, not the full dataset. It was assembled so notebook 1 can actually perform feature engineering from raw inputs without duplicating the full `FAST-Dataset/` tree inside `notebook/`.

In the bundled demo:

- 36 participants were selected;
- all `SUS` positives across Build A and Build B were preserved;
- additional negatives were added so the `SUS binary` demo remains trainable;
- notebook 1 generates Build A and Build B spreadsheets from those raw files.

## Command-Line Usage

Generate the feature-engineered demo spreadsheets:

```powershell
python notebook\scripts\feature_engineering_pipeline.py --config notebook\configs\feature_engineering_fast_sus_demo.json
```

Run the classification demo:

```powershell
python notebook\scripts\classification_pipeline.py --config notebook\configs\classification_config_fast_sus_demo.json
```

If you already have precomputed head metrics and only need schema alignment:

```powershell
python notebook\scripts\transform_head_metrics.py --input my_metrics.xlsx --output notebook\outputs\my_metrics_standardized.xlsx --config notebook\configs\transform_config_template.json
```

## Methodological Rules

- Model selection happens only on the train dataset.
- Resampling strategies are applied only inside training folds.
- Thresholds are selected from training-fold predictions and then transferred to the test set.
- The test dataset is evaluated only once after pipeline selection.

This mirrors the Build A -> Build B logic used in the main FAST experiments.

## Scope

This folder is designed for:

- feature engineering from raw FAST-style score sheets + tracking files;
- train/test classification on generated head-feature spreadsheets;
- demonstration of the benchmark workflow described in the paper.

It does **not**:

- replace the full repository experiment suite;
- reproduce every markdown report and every ablation already present in the root project;
- package the entire FAST dataset inside `notebook/`.
